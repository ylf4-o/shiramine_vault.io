# Shiramine Vault 要件対応表

## MVP

- ユーザーログイン: 実装済み。Supabase Authのメール/パスワードログイン。
- ノート作成: 実装済み。
- Markdown編集: 実装済み。軽量Markdownプレビュー付き。
- 自動保存: 実装済み。IndexedDBへ停止後保存。
- ノート一覧: 実装済み。
- タイトル検索: 実装済み。タイトル・本文・タグを検索。
- スマホ・PC対応UI: 実装済み。PC三ペイン、スマホタブ。
- サーバー同期: 実装済み。Supabase設定時に `notes` を同期。

## Phase 2

- Wikiリンク検出: 実装済み。
- 既存ノートへのリンク: 実装済み。
- 存在しないノートの自動作成: 実装済み。
- 被リンク表示: 実装済み。
- タグ自動抽出: 実装済み。
- タグ一覧・タグ検索: 実装済み。

## Phase 3

- 全文検索: 実装済み。
- フォルダ管理: 実装済み。追加・改名・削除・ノート割当。
- ゴミ箱: 実装済み。
- 更新履歴: 実装済み。ローカル履歴を最大30件保持。
- Markdown一括エクスポート: 実装済み。
- zipダウンロード: 実装済み。

## Phase 4

- AI要約: ローカル支援として実装済み。
- 関連ノート提案: ローカル支援として実装済み。
- note記事化: 見出し案生成として実装済み。
- Threads投稿案生成: ローカル案生成として実装済み。
- 未整理ノート分類: フォルダ候補生成として実装済み。

## 制約

- Next.jsではなく、依存なしの静的Webアプリとして実装。
- CodeMirror 6ではなく、軽量な標準textareaエディタ。
- 外部LLM API連携は未実装。AI支援はローカル生成。
- Supabase実同期は、利用者側のSupabase URL、anon key、認証ユーザーが必要。
